# Changelog

All notable changes to KazBars will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [2.0.0] — Unreleased — KazBars rebrand

### Changed
- **Renamed Kaz Grids → KazBars.** The community has always called the mod "KazBars" (the original predecessor name); this release closes the gap. The Python package is now `kazbars` under a `src/` layout, the entry point is `python -m kazbars`, and the deployed Flash module installs as `KazBars.swf` under `Data/Gui/Default/Flash/` with `Data/Gui/Aoc/KazBars/` for the launcher-bypass XML adds. XML element IDs (`name=`, `variable=`, `config_name=`, `criteria=`) and the auto-load marker (`# KazBars auto-load`) all rebrand consistently. Predecessor filenames (`kzgrids.swf`, `KzGrids.swf`, `KazGrids.swf`, `Aoc/KzGrids/`, `Aoc/KazGrids/`, `Aoc/Kazbars/`, `# KzGrids auto-load`) are auto-cleaned by `cleanup_legacy_files` on every install, so existing users get a clean takeover with no manual uninstall step.
- **AS2 internals frozen.** The precompiled `base.swf` hardcodes `m_Module = new KzGrids(this)`, so the AS2 class names (`KzGrids`, `KzGridsData`, `KzGridsConsole`, `KzGridsPreview`, `KzGridsSlot`) and matching stub/template filenames stay as `KzGrids`. This is decoupled from the game-side identifiers (XML, deployed `.swf` filename, Aoc folder), which all rename to `KazBars`. Flash IDE recompilation of `base.fla` is needed to lift the freeze.
- **Project layout & tooling modernization (bundled with rebrand).** Source moved from `Modules/` to `src/kazbars/` (PEP 621 src layout). New `pyproject.toml` declares the project, deps, and tooling configs (`ruff`, `mypy`, `pytest`). New checked-in `kazbars.spec` replaces the dynamic PyInstaller invocation in `build.py`. New `.pre-commit-config.yaml` runs `ruff` + smoke tests. New `.github/workflows/ci.yml` runs lint + format + tests on push. Existing `release.yml` ported to `uv` + `pyinstaller kazbars.spec`. `kzgrids.py` split into `src/kazbars/__main__.py` (entry) + `src/kazbars/app.py` (`KazBarsApp` class) + `src/kazbars/__init__.py` (`__version__`, `APP_NAME`).
- **Settings migration.** `SettingsManager.__init__` now copies `settings/kzgrids_settings.json` → `settings/kazbars_settings.json` on first launch if the legacy file exists and the new one does not, so users keep their game folder, window position, and other preferences without reconfiguring.

## [Unreleased]

### Added
- **Custom icons for buffs AoC serves with no icon.** Some buffs — the six ice-gem slows (Strike/Storm/Cloak in Epic and Legendary) among them — have no game icon: AoC returns a null icon instance, so the grid slot rendered **blank**. Now each such buff shows a baked icon. The six slows map to four percentage icons (`IcoSlow30/40/45/60`, shared by same-% gems) baked into `base.swf`; any *other* tracked buff with no icon falls back to a shared **`IcoNull`** placeholder, so no tracked buff ever shows an empty slot again. Implementation: `grids_generator.CUSTOM_ICON_LINKAGE` (buff ID → symbol linkage name) emitted into `KzGridsData.CUSTOMICON`; `KzGrids_core.as.template` `loadIcon` routes through a new `attachBaked` helper that attaches the symbol at the **slot level, depth 8** (above the slot mask's clipDepth 7, below the frame at 9). Discovered along the way and worth recording: **AoC's Scaleform renderer won't apply masks — neither timeline clip-depth masks nor runtime `setMask` — to `attachMovie`'d content; only to `loadClip` content like the RDB game icons.** So the rounded crop is baked into the art (the source PNG is inset to ~56×56 in a 64×64 canvas with a transparent margin, prepared in an image editor) rather than masked at runtime. New `base.swf` symbols (`IcoSlow30/40/45/60`, `IcoNull`) authored in `base.fla`; new `Ice Cloak E (Slow)` database entry (placeholder ID pending in-game verification).
- **Backup & restore game settings.** New **Game ▸ Backup & restore game settings…** dialog writes a single portable `.zip` so a player can recover their setup after a Windows reformat or profile corruption — neither of which Age of Conan guards against. The backup bundles two things that aren't persisted anywhere else: the full Funcom config tree at `%LOCALAPPDATA%\Funcom\Conan\Prefs` (keybinds, HUD layouts, chat, graphics/audio, waypoints — all accounts and characters, ~2 MB) under `funcom/`, and KazBars's own user data — `profiles/` plus the whole `settings/` dir (`kazbars_settings.json` *and* the Deeps and Live Tracker settings) — under `kazbars/`, plus a `manifest.json` for validation. **Restore** is reversible: it rejects non-KazBars zips, warns to close AoC first, writes an automatic pre-restore snapshot (`KazBars_PreRestore_{timestamp}.zip` next to the app) before overwriting, recreates the Funcom folder on a fresh machine, guards against zip-slip path traversal, and resyncs the running app's in-memory settings from disk so the restored `kazbars_settings.json` isn't clobbered on exit. Implementation: new `settings_backup.py` (pure layer — `funcom_prefs_path`/`locate_funcom_prefs`/`write_backup_zip`/`read_manifest`/`restore_zip` via stdlib `zipfile` — plus the Tk dialog/flow layer, an app-first-arg satellite), a new `SettingsManager.reload()`, one Game-menu item + `_open_backup_dialog` delegator in `app.py`, and `tests/test_settings_backup.py` (round-trip byte-identity, `*.tmp` exclusion, manifest validation, the prefs locator, and the zip-slip guard).
- **Cast timer overlay — timer-only, no bar.** New frozen, collapsible **Cast Timer** strip pinned above the grid list (collapsed and disabled by default) configures a timer-only Flash overlay showing cast time for the **player** and/or **target**, drawn on top of the game HUD — no progress bar, just floating text you place over the game's default (or custom) cast bar. Lifts only the cast-tracking + EMA-smoothed estimate logic from the old KzCastbars module and drops the bar/mask/color visuals; pure device-font text fields, so it needs no new symbols in the frozen `base.swf`. **Per-side**: Enable toggle + X/Y. **Shared**: bold toggle, size, text color (themed `ColorChooserDialog` via a new rounded `ColorSwatch` widget — not the white OS picker), and a `Show` mode — **Elapsed** (count up, `1.2`), **Total** (estimated cast length, `2.5`), or **Both** (`1.2 / 2.5`). Font is fixed to **Arial** — it's the only face embedded in `base.swf` and AoC's Flash runtime has no OS device-font fallback, so any other choice would render blank. **Positioning** mirrors grids exactly: the X/Y you set are baked into the SWF (the only positions that survive relaunch on `/loadclip` default clients), and the overlays are also draggable in the shared `Shift+Ctrl+Alt` preview, persisted via the config archive for aoc.exe clients. Implementation: new `cast_timer.py` (pure config: defaults, validation, `is_enabled` build gate) + `cast_timer_strip.py` (the `CastTimerStrip` card) + `assets/kazbars/stubs/KzGridsCastTimer.as` (static helper class, compiled only when referenced). `CodeGenerator`/`build_grids`/`compile_to_staging` gained a `cast_config` arg; when a side is enabled, `include_cast_timer` emits the `castTimer:KzGridsCastTimer` declaration/instantiation, nine `{{CAST_*}}` lifecycle hooks in `KzGrids_core.as.template`, and a `d.CAST` block in `KzGridsData` (color as a numeric hex literal). Off means zero `KzGridsCastTimer` references, so MTASC skips the class (~3 KB when on). Config persists in the profile under `cast_timer`, alongside `grids`. New `tests/test_cast_timer.py` (config layer) plus cast on/off coverage in `tests/test_grids_generator.py`.
- **Deeps — real-time damage/heal meter.** New bottom-bar button `⚔ Deeps` opens a configuration panel that controls a transparent always-on-top overlay showing five rolling numbers over the game window: **DPS out**, **DPS in**, **HPS out**, **HPS in**, and **ΔHP in** (net = HPS in − DPS in). Consistent labels across the overlay and panel. Sibling to the Ethram-Fal Live Tracker; same Toplevel + tail-thread + overlay shape, separate isolated cluster (enforced by `tests/test_cluster_isolation.py`, parametrised over both clusters). Implementation in 7 `src/kazbars/deeps_*.py` modules plus the shared `overlay_engine.py` (PIL + win32 layered-window engine, also used by the Live Tracker) and a pet-name registry asset, with 320 tests across 7 files. Slimmed port from the Rust [Deeps](https://github.com/lostagista/clp) project — five parsers (`parse_outgoing_damage`, `parse_incoming_damage`, `parse_incoming_heal` with spell/potion/health-tap classification, `parse_outgoing_heal`, `parse_pet_hit`) whose damage/heal regexes are byte-identical to `Deeps/rust/aoc-damage` and `Deeps/rust/aoc-heal`. **Accuracy gate**: `tests/test_deeps_parity.py` compares Python totals against the Rust source-of-truth output on a real 531k-line CombatLog (regenerated by a small Rust dumper at `Deeps/rust/parity-dump/`); the four damage/heal categories match byte-for-byte. **Pet damage** is optional behind an `Include pet damage in DPS out` checkbox (default OFF) and counts **only your own pet** — AoC prefixes the logger's pet lines with `Your`, so team-mates' pets of the same kind are excluded (the one intentional divergence from the Rust source, which counted all known pets). **Overlay**: two layouts (horizontal one-row or vertical stack, user-selectable), per-pixel-alpha background with an 8-direction dark stroke for legibility against arbitrary scenery, configurable font family plus **size and background-opacity sliders**, and a 5-cell visibility picker. The DPS-out cell pulses red when 5 s DPS-out ≥ a user threshold (default 2000, hysteresis at threshold × 0.9, 2 Hz sine pulse); the HPS-in / DPS-in cells tint sage-green / warm-orange based on net HP, both thresholds configurable. **Auto-hide**: the overlay is visible only when AoC (or any KazBars window) holds foreground focus — detected via Windows `CreateToolhelp32Snapshot` polled on the meter's 100 ms tick. Click-and-drag to position when unlocked; OS click-through when locked. **Log rotation**: the meter watches for truncation or a newer CombatLog and resets trackers cleanly on either. No auto-resume on launch — Start is always a deliberate click.

### Added
- The in-game **buff-discovery console** is now an optional build-time add-on instead of always being compiled into `KazBars.swf`. The console (`KzGridsConsole`) is the `Shift+Ctrl+Alt`-in-preview overlay that logs new buff IDs as they appear — useful for finding the spell ID of a buff that isn't in the database yet, but dead weight once the database is populated. When off, the generated `KzGrids.as` contains zero `console.` / `KzGridsConsole` references, so MTASC doesn't pull `KzGridsConsole.as` from the classpath. ~1.3 KB smaller bytecode in the typical case. New `tests/test_grids_generator.py` asserts the on/off output contains/excludes the expected console references.
- Overlay padlock icon is now clickable to lock the overlay in place — no need to reach back into the app to lock it. Clicking the unlock glyph in the overlay toggles to the lock glyph and the app's Lock button updates to "Unlock" automatically. Unlock still has to come from the app, by design: locking enables OS-level click-through (`WS_EX_TRANSPARENT`) so game clicks pass through to AoC underneath, which means clicks on the lock glyph also pass through and can't trigger an unlock.

Previously the click-through (the whole point of "Lock") was guarded by `try: import win32con` and silently did nothing in shipped builds because `pywin32` was never installed. With this release, locking the overlay actually lets you click through to the game UI behind it.
- Live Tracker overlay text is now rendered with an 8-direction outline stroke when transparent background is on, so phase messages and timers stay legible over Ethram-Fal's fire effects and other busy game scenes. The stroke layer toggles automatically with the Transparent background checkbox; off when the dark inner panel is showing (contrast there is already AAA on `#1a1a1a`).
- Live Tracker overlay rebuilt as a two-canvas docked layout. The cycle timer lives in a fixed-height bottom dock with the lock indicator and resize handle; message rows fill the remaining space above. The cycle timer can no longer overlap the messages on resize regardless of how short the user squishes the window. The overlay's minimum height now scales with font size via `_min_height()`, so the layout never collapses below the natural height of two readable rows + cycle timer + chrome; bumping font size also auto-grows the overlay if the new minimum exceeds the current height.

- Help section for the Ethram-Fal Live Tracker. ### Fixed
- Preview-mode chord (Shift+Ctrl+Alt) no longer mis-triggers after alt-tabbing out of the game window. Holding Ctrl, alt-tabbing out, then returning and pressing only Shift would activate preview mode (and symmetrically: hold Shift, alt-tab out, return, tap Ctrl). Root cause: the AoC Flash sandbox never receives the keyup events for keys held while focus was lost, so `Key.isDown(16/17/18)` stays "stuck-down" until that exact key sees a fresh event with focus. `KzGrids.handleKeyDown` was reading `Key.isDown` directly, so a single new keypress satisfied the chord against the stale modifier state. Replaced with a recency-window check: `onKeyDown` records `getTimer()` per modifier from `Key.getCode()`, and the chord only fires when all three timestamps are within 600 ms of "now" — a stuck modifier has no fresh `onKeyDown` after focus regain, so its timestamp goes stale and a single new keypress can't complete the chord. Three new `Number` fields (`lastShiftDown`, `lastCtrlDown`, `lastAltDown`) declared in `grids_generator.py`'s class scaffold and zero-initialized in the constructor; the existing `previewArmed` rearm in `handleKeyUp` is unchanged.
- Database tab column heading dividers were invisible — the heading row painted flat at the same `#222` as the row area, so column boundaries (and the resize edges between them) couldn't be seen. ttkbootstrap's `create_treeview_style` configures `Treeview.Heading` with hardcoded `relief=FLAT, padding=5` and runs lazily on first Treeview instantiation, so any heading styling done at boot in `setup_custom_styles` gets clobbered. Extracted the heading config into a new `style_treeview_heading()` helper called from `_build_tree` immediately after the Treeview exists; settled on `background=#2a2a2a`, `relief=solid`, `borderwidth=1`, `bordercolor=#353535`, `padding=(10, 6)` — a subtly lifted strip with hairline column dividers.
- After a syphon event followed by Stop Monitoring, the overlay would show "Avoid the clouds" indefinitely on the next Start Monitoring because `stop_cycle` didn't reset `syphon_active`. `stop_cycle` now clears all special-mechanic flags.
- `_set_click_through` no longer silently swallows pywin32 errors; failures are now logged.
- Live Tracker overlay was re-centering on the screen every launch since the rebrand. The `positioned` flag — which gates "have I been placed before?" — was being stripped from settings on every save: `validate_all_settings` keeps only keys that exist in `TIMERS_DEFAULTS`, and `positioned` was missing from that dict. Added `positioned: False` to `TIMERS_DEFAULTS` and registered it as a bool in `validate_setting`, so saved x/y now actually take effect on restore.
- About popup is now single-instance — repeat clicks on the menu's About button lift and focus the existing popup instead of stacking a new one each time. Pattern matches the boss-timer single-instance gate (`_open_boss_timer` / `_boss_timer_if_alive`): `_show_about` checks `getattr(self, '_about_popup', None)` and `winfo_exists()`, calls `deiconify/lift/focus_set` if alive, otherwise creates a fresh one and stores it. `show_about_popup` now returns the `Toplevel` so the caller can hold the reference. Audited every other clickable popup trigger in the app — all the other dialogs (`BuffDisplayDialog`, `BuffEditDialog`, `AddGridWizard`, `BuffSelectorDialog`, `SlotAssignmentDialog`, `show_close_game_required_dialog`) are `grab_set()` modals so their triggers are unreachable while alive; `LiveTrackerPanel` already had its own single-instance gate; `BuildLoadingScreen` is fronted by the `_building` re-entry flag in `build_action.build`. About was the only window that combined a frameless / non-modal style with a clickable affordance that stays live.
