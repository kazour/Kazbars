## [Unreleased]

### Fixed
- About popup is now single-instance — repeat clicks on the menu's About button lift and focus the existing popup instead of stacking a new one each time. Pattern matches the boss-timer single-instance gate (`_open_boss_timer` / `_boss_timer_if_alive`): `_show_about` checks `getattr(self, '_about_popup', None)` and `winfo_exists()`, calls `deiconify/lift/focus_set` if alive, otherwise creates a fresh one and stores it. `show_about_popup` now returns the `Toplevel` so the caller can hold the reference. Audited every other clickable popup trigger in the app — all the other dialogs (`BuffDisplayDialog`, `BuffEditDialog`, `AddGridWizard`, `BuffSelectorDialog`, `SlotAssignmentDialog`, `show_close_game_required_dialog`) are `grab_set()` modals so their triggers are unreachable while alive; `LiveTrackerPanel` already had its own single-instance gate; `BuildLoadingScreen` is fronted by the `_building` re-entry flag in `build_action.build`. About was the only window that combined a frameless / non-modal style with a clickable affordance that stays live.

### Changed
- Source affordance colorized in both Add Grid surfaces. The Player and Target radio labels now carry their type accent (cyan `#3498db` / orange `#e67e22`, bold), the four preset shape buttons in the Add Grid wizard recolor live (`info-outline` ↔ `warning-outline`) when the source flips, and the empty-state preview cells + Custom card's `+` glyph track the same color via the existing `_redraw_empty_cards` path. Implementation note: ttkbootstrap's `tb.Window` strips unrecognised `style=` kwargs at widget construction (parses them as bootstyle keywords), so the two new `Player.TRadiobutton` / `Target.TRadiobutton` styles in `setup_custom_styles` have to be applied via `.configure(style=…)` after the widget exists. Empty-state row-height misalignment also fixed: the 10×1 vertical bar preset card was 200 px tall vs the 130 px squares so it stuck up above the row; dropped to `PRESET_CARD_SQUARE` (130) so the row tops align. Subtle: bumped the Source row's bottom padding from `PAD_XS` (4) to `PAD_LF` (8) so it doesn't crowd the Mode row beneath it.
- Top menu reorg. File menu gains **Exit** (after the separator, wired to `_on_close` so the unsaved-changes prompt still fires). Game menu gains **Change game folder...** and **Uninstall from game client...** (the latter moved out of File). The bottom-bar path-label context menu's "Change game folder…" was respelled to "Change game folder..." so the same command reads identically in both places. `CustomMenuBar` docstring trimmed of a stale "Alt activation" claim that was never implemented.
- Grids excluded from the build now visually mute in the editor list. Toggling Enabled off drains the grid's identity in lock-step: title drops from `heading` (`#FFFFFF`) to `dim_text` (`#888888`), the type badge drops to the same dim, the card frame border falls from the player/target accent to neutral `border` (`#444444`), and the internal accent strip (visible when expanded) follows. The shift is instant per DESIGN.md's *No-Decorative-Motion Rule*; the Enabled toggle stays the affordance, the fade is feedback. Tab focus into a disabled grid lights its title in `muted` rather than full `accent` so the row never looks live mid-tab. Internally: `bind_card_events` accepts a callable for its resting color so the same hover binding survives an enabled→disabled flip without rebinding; `CollapsibleSection` gained a public `set_dimmed()` and stores its accent canvas + content bar as instance state; `GridEditorPanel._apply_enabled_styling` is wired to the Enabled checkbox `command=` and re-applied on init.
- Toast system pass: coalesce, perf, voice, and consistency. **Coalesce by key** — `ToastManager.show` and `app_toast` now accept `key=`; live keyed toasts update in place instead of stacking. The static-grid resize trim/restore pair (and the `1×2` dynamic-coercion follow-up) all share `key='grid_resize'`, so holding the rows/cols spinbox arrow at full assignment produces a single in-place-updating toast instead of ~60 stacked ones over a 6 s burst. **Reposition perf** — hoisted `update_idletasks()` out of the inner `_reposition` loop (one trailing flush instead of one per toast); benefits every toast surface, not just the resize warning. **Entrance animation correctness** — `_animate_entrance` was reading `place_info()['y']` fresh each step and treating that as the target, but its own previous-step `place_configure(y=…)` had already overwritten that value, so the toast drifted progressively off-screen below; pre-coalesce this was masked by the per-tick `_reposition()` re-placing the toast every emit. Fixed by capturing `target_y` once before the step loop. **Brand fit** — toast bg switched from `Surface Sub` (`#1a1a1a`) to `Surface Input` (`#2f2f2f`) per DESIGN.md §5; `wraplength=360` on the label so long buff-name previews no longer clip on narrow windows; fade-out extended from 200 ms (8×25) to 420 ms (14×30) so the warning-yellow accent bar transitions smoothly instead of reading as a flash; resize-toast copy properly singularized ("Unassigned 1 buff" vs "Unassigned 3 buffs"). **API consistency** — every toast in the app now goes through `app_toast(widget, message, style, duration=, key=, on_click=)`; the seven direct `*.toast.show(...)` callers in `build_action.py`, `game_folder.py`, `profile_io.py`, `update_check.py`, and `database_editor.py` (13 sites) were converted, and `DatabaseEditorTab` lost its `toast=` constructor parameter and `self.toast` attribute since the walker pattern resolves the manager from any descendant. Default duration aligned (`ToastManager.show` 4 → 6, matching `app_toast`). `app_toast` gained `on_click=` passthrough for the update-check release-notes toast. **Pre-build auto-save no longer toasts** — `do_save_profile`/`_commit_saved_profile` accept `silent=True`; `build_action` passes it so the "Saved: …" toast doesn't pile up against the "Built — …" toast a few seconds later. **Build success toasts trimmed** — "Built — /reloadui in-game" / "Built — launch the game" / "Built — /reloadui + /reloadgrids" dropped the redundant "Built — " prefix; the success color and the build-loading screen carry the confirmation. **Onboarding step 4 tracks the deployed-SWF state** — the in-panel tip guide's Build step now resets to "not done" the moment any grid edit happens after a successful build (the deployed SWF no longer matches the in-memory state); previously it stayed green until the next build, which gave a misleading "you're done" signal after the user cleared all grids to start over. Step 4 also persists across app restarts: after a successful build, a SHA-1 signature of `{profile_path, grids}` is written to `settings/kzgrids_settings.json` (`last_build_signature`); on profile load the same payload is recomputed and compared, so reopening the app on the same profile with no edits keeps step 4 green without requiring a rebuild. The profile path is part of the signature so cross-profile loads (e.g. building MyGrids then loading Default) correctly leave step 4 inactive even if the grids happen to hash to the same shape after migration; `None` is a valid identity, so building Default itself and later reloading Default also restores correctly.

### Added
- **Default Buff Bars editor** (Game → Default buff bars…). New modal that edits four AoC HUD XML files in place — Player + Target portraits (`CharPortraitLeft/Right.xml`), Top bar (`HUDView.xml`), Floating portraits (`FloatingPortraitView.xml`). One collapsible section per file with the same form: a "Show buff list" toggle that wraps the `<BuffListView />` element in a `<!--KZ_OFF ... KZ_OFF-->` comment span when off (game ignores it, the editor still sees it for re-enable), icon size + spacing spinboxes (8–128 px / 0–50 px, written as `Point(N,N)`), max-columns spinbox (1–30), and a Friendly / Hostile / Friendly + Hostile filter radio group. Writes go only to `Customized/`; the source under `Default/` is read-only. **Mirror-the-XML semantics:** the form blanks any field whose source attr is missing or unrecognised, and Apply only writes attrs the user explicitly set, so a stock file with no `filter` attribute stays clean if the user never picks one — and a custom UI's exotic `filter="all"` survives an icon-size change verbatim. **Surgical regex edits:** the `<BuffListView />` tag is the only thing rewritten; every byte outside it is preserved. Per-attr skip-when-equal keeps the file byte-identical when the user only touches one field. Missing attrs are injected before the closing `/>` (the previous "no-injection" rule meant the user's filter pick never reached stock files). One-shot `*.kzgrids.bak` backup written next to each Customized file the first time the editor touches it. Section open/closed state persists across opens via `settings['buff_display_section_open']`; first-launch default opens Player only (others collapsed). Layout: scrollable body with the bottom Cancel | Apply row pinned (packs `side='bottom'` first so it can't be squeezed at any window size); Apply uses `success` bootstyle for primary affordance; success and failure both go through `app_toast` (failure with `key='buff_apply_failed'` so retries coalesce). Inline "(no filter set in source)" hint appears next to the radios when `filter_var` is empty so all-radios-unselected reads as intentional rather than broken. Custom-UI banner ("Edits stay in your skin.") only renders when `Customized/Views/HUD/` contains files we don't manage. The pre-flight no-game-folder check is the only modal in the module — toast can't render before the dialog exists.

## [1.1.0] — 2026-04-22

### Added
- Background check for new releases on launch. Click the toast to open release notes.
- File → Uninstall from game client. Cleanly removes Kaz Grids files from the selected install.

### Changed
- Grids now reference buffs by spell ID instead of name. Renaming a buff in the database no longer breaks grids that use it. Existing profiles migrate automatically on load; any buffs that can't be resolved are listed in a dialog so you can re-add them.
- About dialog now links to the GitHub repo.

### Fixed
- Database changes are now saved when you close the app and click "Yes" on the unsaved-changes prompt. Previously only the profile was saved.
- Build now refuses to run if an enabled grid has no tracked buffs, instead of installing a silent grid.
- Invalid entries in the Add Buff ID field show a warning before saving instead of being dropped silently.
- Welcome popup no longer references a non-existent "Edit → Clear All Grids" menu path.
