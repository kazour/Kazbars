# Changelog

All notable changes to Kaz Grids will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Changed
- Internal: `profile_io.py` seam refactor. `load_profile` is split into `read_profile_file` (pure I/O, returns `(data, is_corrupt)`) and `apply_profile_data` (dispatch step — grids panel, missing-buff warning, **boss-timer fan-out when the live tracker is open**, reference_resolution, current_profile, settings, title). All four call sites — startup auto-load in `kzgrids.py`, `open_profile`, `load_default_profile`, and the first-launch "Use Defaults" branch — now compose the two explicitly so the boss-timer dispatch is visible from every caller. The `_load_profile` delegator on `KzGridsApp` was removed. `do_save_profile` retained as orchestrator (its error-handling boilerplate would otherwise repeat at every save site); body factored into `build_profile_payload` (the live-tracker pull is named explicitly in its docstring), `write_profile_file`, and `_commit_saved_profile`. Resolves the architectural seam flagged in `docs/architecture-notes.md` Finding 1.
- Internal: Pylance type-checker pass. Two real bugs fixed: `GridEditorPanel.load_from_config` was passing `int` to `StringVar.set` for `x_var`/`y_var` (worked at runtime via tk's auto-coercion, now wrapped in `str()`); `DatabaseEditorTab.__init__`'s `toast=None` default was misleading because the sole caller always passes a `ToastManager` and 14 `self.toast.show(...)` sites assume non-None — `toast` is now a required keyword-only parameter. Remaining diagnostics are false positives from ttkbootstrap monkey-patching `bootstyle` onto `tkinter.ttk` widgets, suppressed inline with `# type: ignore[call-arg]` / `[call-overload]` on the offending lines in `kzgrids.py`, `grids_panel.py`, and `database_editor.py`. Also suppressed `pack_forget` on a `winfo_children()` union narrowed to `Toplevel` — at runtime the children are only frames/widgets.
- Database editor pass: rows now tint by type (debuff in muted red `#F0A0A0`, misc in warm gold `#E0C880`), both AAA-contrast on the dark surface and paired with the Type column label so the encoding survives without color (DESIGN.md). Sortable column headings show ▲/▼ arrows on the active sort, with two trailing spaces reserved on inactive headings so the heading width stays constant when sorting flips. The empty-search state replaces the fake "No matches found" row injected into the tree with a centered `ttk.Label` overlay placed against the tree itself (so the scrollbar doesn't visibly offset it). Five blocking modal warnings (no-selection on edit/delete, empty import file, no-filter-match on export, "All" selected for rename, name-taken on rename, empty rename) are now toasts; a redundant `Messagebox.show_info` after the success toast on import/export was removed. Em dashes purged from user-facing strings (project-wide ban). The "Save Database" button width aligned to `BTN_MEDIUM`; Export tooltip honestly says "currently filtered" instead of "selected".
- Internal: `database_editor.py` cleanup pass — 904 → 866 lines. Centralized post-mutation refresh in `_after_db_change()` (5 callsites: add/edit/delete/import/rename-category). Unified the add/edit dialog validators into one `_make_buff_validator(old_ids=None, old_name=None)` (no args = add-mode, both args = edit-mode with row excluded from collision). Removed dead `BuffDatabase.find_entry_by_id` and the dead `state='normal'` resets in `_on_stacking_changed` (the partial toggle and `stack_start_spin` are never set to `'disabled'`). Hoisted the duplicated `result['stackStart']` write in `_apply_stacking_fields`. Renamed `parse_ids → _parse_ids` (single internal caller). Inlined `buff_type = TYPE_FILTER_MAP.get(...)` that was being shadowed by a per-buff loop variable in `refresh_list`/`export_buffs`. `create_widgets` does `buff = buff or {}` once, collapsing seven `buff.get(k, default) if buff else default` ternaries to plain `.get` calls. `migrate_legacy_buff_fields` no longer returns `buff` (in-place by design; only caller ignored the return). Tree-clear loop replaced with `tree.delete(*tree.get_children())`. Treeview empty-area background and the per-instance `_fix_tree_bg` `after_idle` shim moved into `ui_helpers.setup_custom_styles` so the global ttk style is configured once at app start instead of every tree build.
- Grids panel polish: `_create_from_empty_state` no longer crashes on the Custom preset card (typo fix). Save now guards against transient empty/`-` spinbox text — previously a Ctrl+S during a multi-key spinbox edit would raise `ValueError`. Empty-whitelist label updated to be actionable ("No buffs tracked. Click 'Tracked Buffs...' to add some."). Sort combobox widened so "Shortest first" no longer clips on Windows DPI scales. Empty-state preset cards now reflow into multiple rows on `<Configure>` when the wrapper is too narrow. Slot-counter danger prefix tightened (`!  ` → `! `). Unicode escapes in the panel summary line normalized to lowercase form for consistency with preset labels. No user-facing change in behavior beyond the bug fixes.
- Internal: `grids_panel.py` cleanup pass — dropped six dead `self.` writes (`_x_entry`, `_y_entry`, `_mode_btn`, `_rows_spin`, `_cols_spin`, `_info_row`/`_info_label`) that were assigned but never read; localized `_timer_group`/`_flash_group` to the building method; consolidated the parallel `_preset_cards` list into the existing `_empty_cards` (which already carried `card_w`); reused `bind_label_hover_colors` for the drag-handle Enter/Leave that was hand-rolled; folded `save_to_config`'s name-validation copy into a single `_validate_name()` call; dropped unused widget returns from `_add_str_spin`/`_add_position_entry`/`_add_spinbox`; trimmed verbose docstrings and decorative comment markers. `grids_panel.py`: 1218 → 1186 lines.
- Internal: extracted shared dialog button width and form-input width tokens. New `BTN_DIALOG = 10` (Cancel/Save/Done/Create pairs) and `INPUT_WIDTH_NUM/_TYPE/_FILTER/_SEARCH` (numeric spinboxes, compact dropdowns, filter combos, search entries) replace literal `width=10`/`width=5`/`width=18`/`width=20` repeated across `database_editor.py` and `grid_dialogs.py`. `BuffEditDialog` collapses its dual `_action_label` / "OK or Save" branching into a fixed "Save" button with a `_header_verb` for the dialog header text. Treeview empty area now matches the working surface (`fieldbackground=TK_COLORS['bg']`) so list panels don't show ttkbootstrap's lighter default below the last row.
- Window: minimum size lifted from 900×500 to 950×660 so the bottom bar's status indicators stay readable at the smallest geometry. The bottom bar now packs with `before=self.content_frame`, fixing the bug where vertical-resizing the window let the expand=True content frame eat the docked bar's space. New default geometry on first launch is also 950×660.
- Internal: shaved 128 lines from the four largest modules. `build_loading.py` (914 → 796): the four near-identical popups (`show_close_game_required_dialog`, `show_welcome_popup`, `show_about_popup`, `BuildLoadingScreen._build_summary_ui`) now share three module-level helpers — `_make_popup_shell` (Toplevel + border + scanlines + KAZ GRIDS title), `_draw_close_button` (rect + glow + hover hit-test + Escape/Return binds), `_center_popup`. `grids_panel.py` (1136 → 1128): collapsed `_add_pos_spin`/`_add_dim_spin` into one `_add_str_spin`, inlined the `_on_rows_changed`/`_on_cols_changed` 2-line passthroughs as lambdas, extracted `_attach_panel` for the duplicated separator+pack+drag-bind+reorder-bind sequence shared by `refresh_panels`/`_repack_panels`. `database_editor.py` (891 → 888): folded the 6-line show/hide branches in `_on_stacking_changed` through `getattr(w, method)`. `grid_dialogs.py` (742 → 743): added `_section()` helper for the `LabelFrame + padding + pack` triple in `AddGridWizard.create_widgets`. No user-facing changes.
- Internal: shrank `kzgrids.py` from 741 to 621 lines. The first-launch flow's coordinator (closures + welcome-popup plumbing) moved into `Modules/first_launch.py` as `run_first_launch(app, app_name)`. Resolution scaling moved out of the root window: `parse_resolution()` now lives in `grid_model.py` and `GridsPanel.scale_to_resolution()` owns the grid-coordinate math (no more reaching into `grids_panel.grids` from `kzgrids.py`). Eight dead delegators (`_format_game_path`, `_prompt_aoc_bypass`, `_update_build_state`, `_pulse_game_hint`, `_save_game_path`, `_save_aoc_bypass`, `_warn_missing_buffs`, `_do_save_profile`) were removed once their last internal callers folded into satellites. Hoisted `threading`/`urllib` imports to module top; added `tk.TclError` to the update-checker's swallowed-exceptions tuple to silence shutdown-race tracebacks. Dropped three view-name aliases and a redundant `current_view` reset. No user-facing changes.
- Internal: split `Modules/ui_helpers.py` (1,497 lines, 8 responsibilities) into six focused modules: `ui_helpers` (design tokens), `ui_widgets`, `ui_components`, `ui_tk_style`, `window_position`, `settings_manager`. Imports inside `Modules/` are now consistently relative. No user-facing changes.

## [1.1.0] — 2026-04-22

### Added
- Background check for new releases on launch. Click the toast to open release notes.
- File → Uninstall from game client. Cleanly removes Kaz Grids files from the selected install.

### Changed
- Grids now reference buffs by spell ID instead of name. Renaming a buff in the database no longer breaks grids that use it. Existing profiles migrate automatically on load; any buffs that can't be resolved are listed in a dialog so you can re-add them.
- About dialog now links to the GitHub repo.

### Fixed
- Database changes are now saved when you close the app and click "Yes" on the unsaved-changes prompt. Previously only the profile was saved.
- Build now refuses to run if an enabled grid has no tracked buffs, instead of installing a silent grid.
- Invalid entries in the Add Buff ID field show a warning before saving instead of being dropped silently.
- Welcome popup no longer references a non-existent "Edit → Clear All Grids" menu path.
